

## خطة جديدة: فصل اختيار الملف عن الدرج بالكامل

### المشكلة الجذرية
كل محاولات منع إغلاق الـ Sheet/Drawer فشلت لأن Radix DismissableLayer يغلق المكوّن عند فقدان التركيز في Android WebView، بغض النظر عن أي guard. الحل الصحيح هو **عدم ربط اختيار الملف بالدرج أصلاً**.

### الحل المقترح (كما اقترحت)

```text
┌─────────────────────────┐
│  زر + في صفحة الوثائق   │
└──────────┬──────────────┘
           │ click → يفتح file picker مباشرة
           │ (بدون أي Sheet/Drawer مفتوح)
           ▼
┌─────────────────────────┐
│  المستخدم يختار ملف      │
│  من منتقي أندرويد        │
└──────────┬──────────────┘
           │ onChange fires
           ▼
┌─────────────────────────┐
│  شاشة رفع كاملة (overlay)│
│  - Progress bar للرفع    │
│  - بعد النجاح: بريفيو    │
│  - حقول الميتا (اسم،     │
│    تصنيف، ملاحظة، تاريخ) │
│  - زر "إضافة" / "إلغاء"  │
└──────────┬──────────────┘
           │
    ┌──────┴──────┐
    │             │
  إضافة        إلغاء
    │             │
  حفظ في DB    حذف من Storage
  + كاش محلي   + عودة للصفحة
    │
    ▼
  عودة لصفحة الوثائق
  مع الملف ظاهر
```

### التعديلات

#### 1. `src/pages/Documents.tsx` - تبسيط جذري

**إزالة:**
- كل منطق `isPickingFileRef` و `primeFilePickerLock` و `schedulePickerUnlock`
- Sheet الإضافة و Sheet التعديل (للملفات فقط)
- الـ `flushSync` import

**تغيير زر FAB (+):**
- بدلاً من فتح Sheet، يفتح file picker مباشرة عبر `input.click()`
- الـ input يبقى خارج أي modal (كما هو حالياً في السطر 1042)

**إضافة حالة overlay جديدة:**
- `uploadingFile: { file: File, progress: number, url?: string, storagePath?: string } | null`
- عندما يختار المستخدم ملف → يظهر overlay كامل الشاشة (div عادي، ليس Sheet ولا Drawer)
- يعرض progress bar أثناء الرفع
- بعد النجاح يعرض بريفيو (صورة أو أيقونة PDF) + حقول الميتا
- زر "إضافة" يحفظ في DB + كاش محلي
- زر "إلغاء" يحذف الملف من Storage ويغلق

#### 2. Overlay الرفع (داخل نفس الملف)

مكوّن `UploadOverlay` بسيط:
- **مرحلة الرفع**: spinner + progress bar + اسم الملف
- **مرحلة المعاينة**: بريفيو الصورة/PDF + نموذج (اسم، تصنيف، ملاحظة، تاريخ انتهاء، تذكير)
- **الأزرار**: "إضافة للوثائق" / "إلغاء"
- هذا div عادي بـ `fixed inset-0 z-50` — لا يستخدم أي مكتبة modal

#### 3. التعديل على زر التعديل (Edit)

- زر "إضافة مرفق" في Sheet التعديل يُغلق الـ Sheet أولاً ثم يفتح file picker
- بعد اختيار وتأكيد الملف في الـ overlay، يتم ربطه بالمستند مباشرة عبر `addDocFileMut`

#### 4. عرض الملفات من الكاش المحلي

- عند عرض مرفق، يتحقق أولاً من `documents-cache-v1` في Cache API
- إذا وُجد يعرض من الكاش مباشرة (أوفلاين)
- إذا لم يُوجد يستخدم `useMediaUrl` مع `bucket: "documents"`

### ملاحظات تقنية
- الـ `<input type="file">` يبقى في مكانه الحالي (خارج كل modal) — السطر 1042
- لا حاجة لأي تعديل على `useMediaUrl` (تم تعديله سابقاً)
- الـ overlay ليس route جديد، هو مجرد `fixed div` يظهر ويختفي بـ state
